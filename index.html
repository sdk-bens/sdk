New UI For GPU SWE
Faster than ever

Working within the GPU SWE infrastructure team, we dealt with extensive data processing and management. The processed results were displayed on a website for our clients, other teams within the GPU SWE department. However, the website pages suffered from slow loading times due to the large amount of data being queried from various databases, including calculations, processes, and results. As part of my projects, I focused on resolving this issue, aiming to improve the speed of data access. The outcome was remarkable, with engineers now able to retrieve the data in less than 100 milliseconds, a significant improvement from the previous waiting time of over one minute.

March 2023

New API for GPU performance results

Over 280 Engineers use the GPU SWE infrastructure system on a daily bases. They query data from a massive SQL Database (over 50 tables), with more than 6 billions row.
Engineers at different teams have different needs for data but the only option available is to use the general api which query all the data.
Teams then have to write scripts to only extract the needed data that meets their needs, therefore spending a lot of time just to deal with the massive data payload.
With the new API, enginners now can query the exact data they need, saving days of not needed extra work.

Disclosure Reinforcemet

Implemented enhanced disclosure guidelines within the GPU SWE infrastructure, ensuring improved data access control. Previously, teams had the ability to query all data, even for projects that were not disclosed. However, with the new system, engineers and teams are granted access only to the data that is directly related to the disclosed projects. This measure strengthens data privacy and security, providing a more focused and restricted approach to data retrieval and usage.
